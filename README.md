# Производственная практика по MLOps

## Цель задания

Развернуть полный MLOps-стек для управления ML-моделями, включая:
- MLFlow (трекинг экспериментов, хранение моделей),
- PostgreSQL (база метаданных MLFlow),
- MinIO (S3-совместимое хранилище артефактов),
- FastAPI (REST-сервис для инференса),
- Seldon MLServer (оптимизированное обслуживание моделей, опционально).


## Было сделано

MLFlow, MinIO, PostgreSQL запускаются с помощью docker-compose. Были открыты порты:
- MLFlow (5000) для веб консоли;
- MinIO (9001) для веб консоли.

В docker-compose добавлена одноразовая задача под названием minio-create-bucket для создания бакета в S3-хранилище MinIO.

Была обучена нейронная сеть с помощью фреймворка PyTorch для классификации ирисов Фишера. В качестве обучающих данных был использован датасет ирисов Фишера из библеотеки scikit-learn. Архитектура нейросети:
- 4 входа;
- линейный слой с функцией активации ReLU и 50-ю выходами;
- линейный слой с функцией активации ReLU, 50 входами и 3-мя выходами, на каждый из классифицируемых классов.

Выход нейросети - чистые логиты, которые в сервисе FastAPI прогоняются через softmax для получения распределения вероятностей принадлежности конкретного ириса конкретному классу.

Был написан сервис FastAPI для удобного заимодествия с моделью c одним endpoint "/predict" и логгированием.
На вход подаётся JSON формата:
```json
{
	"x1": float,
	"x2": float,
	"x3": float,
	"x4": float
}
``` 
, где каждый "xi" - значение конкретного признака. На выходе число - номер конкртного класса ириса (1..3), которому по мнению нейросети он принадлежит.

Модели сохраняются через mlflow в minio (артефакты) и postgres (метаданные), а fastapi сервис скачивает конкретную версию модели с помощью `mlflow.pyfunc.load_model`.

Seldon ML сервер не был использован.

## Перед запуском FastAPI

Если запуск тестовый то можете пропускать этот пункт. Будет использован готовый testing.env.

Необходимо подправить файл .env, чтобы инфраструктура могла корректно запуститься.

Что надо изменить/добавить в .env для docker compose:
```
POSTGRES_USER=
POSTGRES_PASSWORD=
POSTGRES_DB=
MINIO_ROOT_USER=
MINIO_ROOT_PASSWORD=
ARTIFACTS_BUCKET_NAME= 
```
ARTIFACTS_BUCKET_NAME - название "подхранилища" в minio для хранения артефайтов модели.

Обязательно поднимите инфраструктуру и обучите модель (о чём ниже).

## Создание инфраструктуры

Для поднятия инфраструктуры используйте `docker compose up`. Будет использован созданный вами файл `.env`.

Если хотите использовать готовый .env (ТОЛЬКО ДЛЯ ТЕСТОВ И РАЗРАБОТКИ), то используйте `docker compose --env-file "testing.env" up`.

## Обучение модели
Обучение модели происходит в файлике `src/experiment1.py`. Для начала процесса достаточно запустить его с помощью `uv run src/experiment1.py`. После завершения процесса обучения соответствующий эксперимент и версия модели добавятся в mlflow.

## Как запустить FastAPI

1. Скачать репозиторий и перейти в папку проекта;
2. Установить пакетный менеджер uv;
3. Подтануть все зависимости: `uv sync`;
4. Войти в созданное виртуальное окружение:
	- Windows: `.venv\Scripts\activate`;
	- Linux/macOS: `source .venv/bin/activate`;
5. Выполнить в терминале (консоли) из корня проекта:
	- Для прода: `fastapi run api/api.py`;
	- Для разработки: `fastapi dev api/api.py`;
6. API доступно на `localhost:8000`;
7. Наслаждаться :D 

## Пример запроса к FastAPI

После запуска сервиса документация доступна в Swagger UI:
- `http://localhost:8000/docs`

Пример тела запроса для `/predict`:

```json
{
	"x1": 5.1,
	"x2": 3.5,
	"x3": 1.4,
	"x4": 0.2
}
```

Пример ответа:

```json
{
	"iris_class": 1
}
```
